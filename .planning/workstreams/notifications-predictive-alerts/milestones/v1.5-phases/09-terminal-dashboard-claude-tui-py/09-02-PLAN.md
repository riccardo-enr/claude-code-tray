when the daemon is down.

    without it uv's cold-run "Installed N packages" line prints into the first frame. Do NOT
    request textual's `syntax` extra -- it pulls the whole tree-sitter set for nothing here.

    Module docstring (triple-quoted, codedoc house style) stating the boundary in both

    Define `ClaudeTui(App)` with:

    `CSS` as an inline class-level string, not a separate stylesheet file. Rules:
    `Screen` lays out vertical; `#body` takes `height: 1fr`; `#usage` and `#trends` take
    `height: auto` with a bottom border so the three panels read as separate blocks (D-01);
    `#sessions` takes `height: 1fr` inside `#body` so the table absorbs the remaining space
    and scrolls internally; `#body.stale` sets `text-opacity: 60%` for D-10's dimming; and

    snapshot, starting `None`, and the epoch of the last successful query, starting `None`.
    The snapshot being `None` is exactly the D-11 cold-start predicate.

    `compose()` yielding, in order: `Header(show_clock=True)` -- which carries D-02's title and
    clock, and whose `sub_title` is the single place freshness is communicated; a
    `Vertical(id="body")` containing `Static(id="usage", markup=False)`,
    `Static(id="trends", markup=False)` and `DataTable(id="sessions")`; then
    `Static(id="coldstart", markup=False)` as a sibling of `#body`; then `Footer()`.
    Every `Static` is constructed with `markup=False` -- `Static.update()` respects that flag,
    and the burn/countdown strings are formatter output that would otherwise be parsed as
    markup. `#coldstart` sits outside `#body` so the cold-start message is never dimmed by
    the `.stale` class.

    `on_mount()`: define the DataTable's columns once with labels `status`, `project`, `time`
    (D-03's three columns), set its cursor type to none since D-01 specifies no navigation,
    then register the two timers via `self.set_interval` -- `core.TUI_FETCH_INTERVAL` driving
    `self.fetch` (D-08) and `core.TUI_TICK_INTERVAL` driving `self.tick` (D-09) -- and call
    `self.fetch()` once immediately so frame one is not blank for two seconds. Set the
    header's `sub_title` to the initial connecting state.

    `fetch()` decorated `@work(thread=True, exclusive=True, exit_on_error=False)`. Its whole
    body is a `try` calling `core.query_snapshot()` and marshaling the result back with
    `self.call_from_thread(self.apply_snapshot, snap)`, and a blanket `except Exception` that
    marshals `self.call_from_thread(self.mark_stale)`. Both halves of the containment are
    mandatory and neither is redundant: `exit_on_error=False` closes the worker door and the
    blanket `except` closes the callback door, and Pitfall 1 documents that the framework
    default on both is to exit the app. Carry a `# ponytail:` marker naming this as the same
    posture `poll_loop` / `serve` / `_handle_conn` already use, and naming D-12's contract --
    retry forever on the same interval, no backoff, no failure cap, no exit. A thread worker
    must never touch a widget directly; every mutation goes through `call_from_thread`.

    `apply_snapshot(snap)`: rebind the snapshot attribute and the last-success epoch in single
    assignments, hide `#coldstart` and show `#body`, clear the `.stale` class from `#body`,
    set the header `sub_title` to the live indicator, then call `render_all()`. The single-
    assignment rebind is what makes a coincident 1s tick safe -- it observes either the old
    snapshot or the new one, never a half-applied one.

    `mark_stale()`: when the snapshot attribute is still `None` this is D-11's cold start --
    show `#coldstart`, hide `#body`, and update `#coldstart` with a single clear message
    saying the daemon is not running and naming `just start` as the fix. Never a traceback.
    Otherwise this is D-10 -- leave the snapshot bound and on screen, add `.stale` to `#body`
    to dim it, and set the header `sub_title` to the unreachable indicator including the last
    successful update rendered as a local `HH:MM:SS` clock time. Do not clear the snapshot,
    do not count failures, do not schedule anything: the 2s timer simply comes round again
    (D-12), and the next success clears the class silently.

    `tick()`: call `render_all()` inside a `try`, with an `except Exception` that sets the
    header `sub_title` to a render-error marker rather than passing silently -- this callback
    also runs under `Timer._tick`, so an unguarded raise here exits the app just as a fetch
    failure would. It must not be a silent swallow: a frozen frame under a live header clock
    would be the stale-as-live failure this plan's first prohibition forbids.

    `render_all()`: return immediately when no snapshot is bound. Otherwise take `time.time()`
    once, then push `core.tui_usage_rows(snapshot usage, now)` newline-joined into `#usage`
    (TUI-01), `core.trend_text(snapshot trends)` into `#trends` (TUI-02, D-05, D-07), and
    rebuild `#sessions`: `clear()` the table -- which preserves the column definitions -- then
    iterate `core.sess_rows(snapshot sessions or [], now)` and add each row. There must be
    exactly ONE `add_row` call site in the file and it must wrap every cell in
    `rich.text.Text`: `DataTable` runs any plain `str` cell through `Text.from_markup` with no
    per-widget opt-out, so a project directory whose name is textual markup is injection, and
    a directory name that is invalid markup raises `MarkupError` inside a timer callback,
    which per Pitfall 1 exits the app. A `Text` instance takes the widget's renderable
    passthrough branch and never reaches the markup parser. This is the same mitigation shape
    as the v1.3 Pango lesson (threat T-05-04) and the v1.4 `textContent` panel
    (`dashboard.py:468-469`, threat T-07-01). Read every snapshot key with `.get(...)`, never
    subscript. Do not use `recompose=True` on the sessions panel: it destroys and rebuilds
    child widgets every second and loses scroll position.

    Do not recompute anything. No trend function is called here, no sparkline is built, no
    second glyph set is introduced -- D-05 and D-06 both. Do not open
    `usage-history.jsonl` and do not run the usage CLI: the socket snapshot is the only data
    source (D-04). Do not send anything on the socket other than the snapshot query.

    Close with the standard `if __name__ == "__main__":` guard constructing and running the app.

    ASCII only in every line of this file. The sparkline glyphs are not typed here -- they
    arrive inside the trend row strings the daemon already built (D-06).
