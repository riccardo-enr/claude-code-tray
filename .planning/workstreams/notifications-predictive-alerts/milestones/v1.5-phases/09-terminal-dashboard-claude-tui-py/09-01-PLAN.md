<objective>
Build the entire pure, textual-free substrate the terminal dashboard renders, inside
`claude_monitor/core.py`, and assert all of it in the `--selfcheck` suite.

`just selfcheck` runs on an interpreter that cannot have textual (PEP 668
externally-managed system python). Everything worth asserting therefore has to live
*above* that boundary. This plan puts the socket client, the usage rows (TUI-01), the
trend text (TUI-02 / D-05 / D-07), the session rows (TUI-03 / D-03 / D-09) and the
refresh timing constants (D-08 / D-09) in `core.py`, where `--selfcheck` can prove them.
Plan 09-02's `claude-tui.py` then contains only the App class and its CSS.

Purpose: the green gate every change in this repo must keep passing is
`just selfcheck` (exit 0). Logic placed above the textual boundary is provable there;
logic placed inside the App is not provable at all. This is the phase's load-bearing
architectural split.

Output: ~12 new pure symbols in `claude_monitor/core.py` and a matching assert block in
`claude_monitor/test_claude_monitor.py`.
</objective>
