## The gap (CR-01)

`Monitor._pop_stale` (claude-monitor.py ~427) pops a stale session dict with no memory of its
prior status. `core.session_stale` deliberately lets an `alive=True` session age out past
REAP_MAX_AGE (this is REQUIRED for the same-pane `/exit`//`clear` self-heal -- 07-02
must_haves.truths[1] -- where the pane stays alive but the session is dead). But when a session
that is GENUINELY still alive idles in `waiting` for over an hour and then sends its next routine
`waiting` resend, `handle` does `s = self.sessions.setdefault(sid, {})` -> a fresh empty dict ->
`old = s.get("status")` reads `None`, `sess_should_notify(None, "waiting")` returns `True`, and
`emit_notif` re-fires the full popup (the fresh dict also resets `acked`, re-arming the `!`
badge) -- for a status that never changed. That regresses NOTIF-02 (Phase 5).

## Fix direction (one option is a trap -- rejected explicitly)

- VIABLE (this plan): short-lived reaped-status memory. `_pop_stale` remembers each popped
  session's last status in `self._reaped_status`; `handle` resolves its notification baseline
  via `core.sess_notify_baseline(live_status, reaped_status)`, so a same-status resurrection
  reads as no transition and does NOT re-notify, while the reap's cleanup and both 07-02
  self-heal paths stay intact.
- REJECTED TRAP (do NOT implement): "narrow the age ceiling so `alive=True` sessions are
  excluded from age-based reap." After `/exit` or `/clear` in the SAME tmux pane the pane is
  still alive (`pane_alive` returns `True`), so excluding `alive=True` from the age reap means
  that case NEVER self-heals -- re-breaking 07-02 must_haves.truths[1], a delivered G-07-2
  guarantee. The executor must not drift into this; `session_stale` is left byte-for-byte
  unchanged by this plan.
