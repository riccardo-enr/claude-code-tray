# Requirements Archive — claude-code-tray v1.5 (TUI Dashboard)

**Status:** ✅ SHIPPED 2026-07-24 — 8/8 requirements delivered.

Milestone goal: surface usage/quota/trends and live sessions in a terminal UI, so
people who live in the terminal get the tray's data without a browser round-trip.
Closes SEED-007. A new `claude-tui.py` entry point became a third consumer of
`claude_monitor.core` alongside `claude-monitor.py` and `dashboard.py`.

Constraints held: X11 host only, no new polling (the TUI queries the existing
daemon rather than shelling out to the usage CLI). `textual` was the one exception
to the stdlib+PyGObject-only rule, scoped to this entry point via an optional extra.

## v1.5 Requirements

### TUI (terminal rendering)

- [x] **TUI-01**: `claude-tui.py` shows current usage (%, tokens, reset countdown, burn rate) for both the 5-hour and 7-day caps. — **Validated**
- [x] **TUI-02**: Shows trends — sparkline, daily/weekly burn, peak-usage hour — reusing `claude_monitor.core`'s existing trend functions, not reimplementing them. — **Validated** (`core.trend_text` joins the snapshot's trend rows verbatim, never recomputed)
- [x] **TUI-03**: Shows a live sessions panel — project dir, status, time-in-state — sorted waiting -> running -> done, matching the v1.4 dashboard panel's semantics. — **Validated**
- [x] **TUI-04**: Refreshes automatically on an interval; no manual re-run or keypress needed. — **Validated** (two-timer loop: fetch + tick)
- [x] **TUI-05**: Degrades cleanly when the daemon isn't running or the socket is unreachable — a clear message, never a crash or an unhandled traceback. — **Validated** (double containment; post-UAT WR-03 routed render failures through the tick guard)

### Socket query verb (SOCK)

- [x] **SOCK-01**: The daemon's existing unix socket gains a read-only query verb returning a JSON snapshot of `self.sessions` plus the last polled usage/history state. — **Validated**
- [x] **SOCK-02**: The query path shares the socket without disrupting the fire-and-forget hook-event path — a stalled/malformed query connection cannot block or corrupt a session-event write. — **Validated** (thread-per-connection `_handle_conn`)
- [x] **SOCK-03**: The query responder reads `self.sessions` safely against the Gtk-main-thread mutator — no torn/partial reads. — **Validated** (`sessions_lock` wraps every call site)

## Deferred (to future milestones)

- Click-to-focus a pane from the TUI (reusing `pane`/`tmux` fields) — the tray remains the click-to-focus surface for v1.5.
- Standalone (no-daemon) mode reading `usage-history.jsonl` directly — shared-socket was chosen to get live sessions in scope; a fallback mode could still be added later.
- TUI polish (color-by-threshold, progress bars, richer trends, optional pane focus) — proposed for v1.6.

## Out of Scope (unchanged)

- Any TUI-side write/mutation of daemon state (mute toggles, config changes) — read-only for v1.5.
- `curses` as the rendering layer — decided against in favor of `textual`.
- Wayland support — the tray itself remains X11-only.
