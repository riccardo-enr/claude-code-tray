## Cross-Phase Integration

The subsystem converges on one choke point, so there is no wiring gap between the producers
(Phase 05) and the config gate (Phase 06):

- `NOTIF_KEYS` maps all four kinds (`waiting`/`done`/`5h`/`7d`) to config keys (claude-monitor.py:77).
- `notif_allowed(kind, config)` enforces mute-wins + per-event gate (claude-monitor.py:132).
- `emit_notif` gates every emission on `notif_allowed` (claude-monitor.py:1597), and is the sole
  emit path for both session events (`Monitor.handle`, line 1861) and predictive cap alerts
  (`poll_loop`, line 1976).
- CFG-05 badge threshold is read from `config["usage_threshold"]` in `build_label` (claude-monitor.py:1766).

## End-to-End Flows

1. **Session needs input / finished -> desktop notification -> click focuses pane** — PASS
   (05-UAT SC1/SC2, live user-confirmed).
2. **Cap projected to exhaust before reset -> single predictive alert, re-arms next window** — PASS
   (05-UAT SC3/SC4, --selfcheck asserts).
3. **User toggles an event / mutes all / sets badge threshold -> takes effect without restart,
   persists across restarts** — PASS (06-UAT tests 1-3 + 4-6, live + inspection).
