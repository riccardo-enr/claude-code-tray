<objective>
Add the thread-safety foundation Phase 8's query verb needs: a `threading.Lock` guarding every
existing `self.sessions` access site (SOCK-03), and a pure, testable `core.build_session_snapshot()`
function that both the existing dashboard write path and the new (Plan 08-02) socket query path
share for building the sessions payload (groundwork for SOCK-01's response shape).

Purpose: today `self.sessions` is single-mutator (Gtk thread only) with one lock-free off-thread
reader (`write_dashboard`, accepted-risk per Phase 7's D-08). Phase 8 adds concurrent query-thread
readers (Plan 08-02), which raises the bar from "accepted-risk" to "no torn/partial reads" (SOCK-03).
This plan delivers that guarantee and the shared snapshot shape before any socket/threading change
lands, so Plan 08-02 only has to wire the protocol, not re-derive the locking model.
Output: `Monitor.sessions_lock`, all four existing call sites wrapped, `core.build_session_snapshot`,
and locking `--selfcheck` asserts.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-01 | Tampering (race condition / data-integrity) | `self.sessions` read/mutate call sites | medium | mitigate | `sessions_lock` wraps every existing call site's full read-modify-write or snapshot-build step as one atomic unit (Task 2), preventing a torn/half-updated session dict from ever being observed by a concurrent reader (delivered by this plan; exercised end-to-end once Plan 08-02 adds the concurrent query reader). |
| T-08-02 | Denial of Service (lock contention) | `reap_stale()`'s `pane_alive`/`session_stale` calls | low | mitigate | The lock scope in `reap_stale()` is deliberately narrowed to the `list(self.sessions.items())` copy only -- the tmux subprocess shell-out (`pane_alive`) and the reap heuristic (`session_stale`) run OUTSIDE the lock, so a slow/hung tmux call never blocks `handle()`, `write_dashboard()`, or a query thread. |

No package-manager installs in this plan (stdlib only; `threading` already imported) -- no supply-chain (T-08-SC) threat applies.
