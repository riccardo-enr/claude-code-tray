# Phase 04 — UI Design Contract

> Visual and interaction contract for the self-contained usage dashboard HTML. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** This is NOT a web-app UI. The artifact is a SINGLE self-contained static `.html` opened via `file://`, generated by Python stdlib (`render_dashboard`) with inline `<style>` + inline `<script>` only. No server, no CDN, no external assets, no JS charting library, no CSS framework (locked D-01/D-02/D-06 — a framework or component library would VIOLATE DASH-06 self-containment). All values below are concrete inline-CSS the executor drops directly into the generated `<style>`/SVG. Charts are hand-drawn inline SVG via `document.createElementNS`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (stdlib-generated inline CSS + hand-drawn SVG) |
| Preset | not applicable |
| Component library | none — no CDN, no dep, self-contained single file (DASH-06) |
| Icon library | none — no icons this phase (charts + text labels only) |
| Font | system stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif`; numeric labels add `font-variant-numeric: tabular-nums` for aligned tick/legend digits |

**Layout:** single centered column, `max-width: 880px`, `margin: 0 auto`, page padding 24px. Three stacked chart cards in DASH order: (1) usage-% trend + range toggle, (2) daily burn-rate trend, (3) hour x day heatmap. Page title row on top. Fully responsive-degrading via SVG `viewBox` + `width:100%` (no media queries required).

**Focal point (Dimension 2):** the **usage-% trend card (card 1)** is the visual anchor — it sits top-of-stack directly under the title, is the only card with the range toggle, and carries the accent-colored trend stroke. The executor should treat it as primary; the burn-rate and heatmap cards are secondary/supporting and never out-weight it visually.

## Spacing Scale

Declared values (subset of the 8-point scale — multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Heatmap cell gap, inline label gaps, toggle-button icon/text gap |
| sm | 8px | Range-toggle button padding (y), gap between axis label and axis |
| md | 16px | Card inner padding, gap between chart title and chart body |
| lg | 24px | Page padding, vertical gap between chart cards |
| xl | 32px | Page title bottom margin / header-to-first-card gap |

Exceptions: SVG-internal geometry (chart plot insets, cell size 24px, axis-tick lengths) is drawing math inside a fixed `viewBox`, not layout spacing, so it is exempt from the 4px grid. Heatmap cell size = 24px square (a multiple of 4).

## Typography

Exactly 3 sizes, 2 weights (400 regular, 600 semibold).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Heading (page title + chart card titles) | 20px | 600 | 1.2 |
| Body / control label (range-toggle buttons, legend text, empty-state message) | 14px | 400 | 1.5 |
| Small (SVG axis ticks, heatmap row/col labels, meta "generated at" line) | 12px | 400 | 1.4 |

SVG text uses the same family; set `font-size` in px attributes matching the scale above (12px for ticks/labels, no 10px "cram" sizes — keep legible at typical screen zoom).

## Color

60/30/10 split, single-hue-210 family (matches the locked heatmap hue for one coherent system).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(210, 20%, 98%)` | Page background |
| Secondary (30%) | `#ffffff` cards on `hsl(210, 16%, 90%)` 1px borders, `border-radius: 8px` | Chart cards / surfaces |
| Accent (10%) | `hsl(210, 80%, 45%)` | Usage-trend + burn-trend line strokes (2px), active range-toggle button fill, focus ring |
| Text foreground | `hsl(210, 15%, 20%)` | Titles, labels |
| Text muted | `hsl(210, 10%, 45%)` | Axis ticks, meta line, inactive toggle text |
| Grid / axis lines | `hsl(210, 16%, 88%)` | Chart baselines, gridlines, cell borders |
| Destructive | not applicable | No destructive actions in this phase |

**Accent reserved for:** the two trend-chart line strokes, the ACTIVE range-toggle button (accent fill + white text), and the keyboard focus ring. Nothing else uses accent — card borders, gridlines, and axis text stay in the neutral 210 grays.

### Heatmap color ramp (LOCKED — D-07)

| State | Value | Rule |
|-------|-------|------|
| Low value | `hsl(210, 80%, 92%)` | Lightness anchor for the minimum non-null cell |
| High value | `hsl(210, 80%, 30%)` | Lightness anchor for the maximum non-null cell |
| Interpolation | `hsl(210, 80%, L%)` where `L = 92 - (v / vmax) * (92 - 30)` | Single-hue light->dark, normalized to the grid's max non-null value |
| Empty bucket | `hsl(0, 0%, 88%)` | Distinct NEUTRAL GRAY — a `None` bucket (no samples) NEVER renders as the low-value blue. "No data" must read as visually distinct from "low usage" (mirrors Phase 3 D-03 gap handling). |

A single-hue sequential ramp is colorblind-safe (lightness-encoded, not hue-encoded). Include a small legend: a low->high gradient swatch labeled with `fmt_tokens`-style min/max tok/hr, plus a separate gray swatch labeled "no data".

## Interaction Contract

### Range toggle (DASH-02 / D-03 — client-side)

Three buttons: `day` / `week` / `all`, each carrying `data-range`. Click filters the one embedded `D.usage` dataset client-side (`t >= D.bounds.day`, `t >= D.bounds.week`, or all) and redraws `#usage-chart`. No regeneration, no per-range pre-render.

| State | Style |
|-------|-------|
| Default (inactive) | transparent bg, `1px solid hsl(210,16%,90%)`, text `hsl(210,10%,45%)`, `border-radius: 6px`, padding 8px 16px |
| Hover | bg `hsl(210, 20%, 96%)`, text foreground |
| Active | bg accent `hsl(210,80%,45%)`, white text, no border shift (one active at a time) |
| Focus (keyboard) | `outline: 2px solid hsl(210,80%,45%); outline-offset: 2px` |

Default active range on load: `all` (full retained history — the dashboard's reason to exist over the tray sparkline). Buttons are real `<button>` elements (keyboard-focusable, Enter/Space activate for free — no custom key handling needed).

### Hover / tooltip

Minimal, native only (ponytail — no custom JS tooltip layer). Each heatmap `<rect>` and each trend data region gets a native SVG `<title>` child (e.g. `"Mon 15:00 — 14.2k tok/hr"`, or `"no data"` for empty cells) so the browser shows a hover tooltip with zero extra JS. No custom-positioned div tooltips this phase.

### Chart line breaks

Both trend polylines break (do not interpolate across) `null` gaps: `burn_series` emits `None` for empty days, `heatmap_buckets` emits `None` for empty cells. The client draws a break/gray, never a false zero — consistent with Phase 3's gap posture.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Page title | `Claude Code — Usage Dashboard` |
| Chart 1 title | `Usage %` |
| Chart 2 title | `Burn rate (daily)` |
| Chart 3 title | `Peak usage — hour x day` |
| Range toggle labels | `Day` / `Week` / `All` |

| Empty state heading | `Collecting usage history...` |
| Empty state body | `Charts appear once the tray has recorded enough usage samples. Leave the monitor running and reopen this page in a few minutes.` |

The primary user action ("CTA") is the tray-side **Open Usage Dashboard** menu item (defined in the plan, not in-page); the page has no submit/CTA of its own — the range toggle is its only interactive control.

## Accessibility

- **Colorblind-safe:** single-hue lightness ramp for the heatmap (no red/green encoding); trends distinguished by position + `<title>`, not color alone.
- **Contrast:** foreground `hsl(210,15%,20%)` on `#ffffff`/`hsl(210,20%,98%)` and white-on-accent active button both exceed WCAG AA (>= 4.5:1). Muted text is reserved for non-essential ticks/meta.
- **Keyboard:** range controls are native `<button>`s — focusable and operable without a mouse; visible focus ring in accent.
- **Legibility:** minimum 12px text at typical screen zoom; SVG uses `viewBox` + `width:100%` so charts scale without pixelation; empty cells and no-data legend prevent "gray = zero" misreads.
- **Semantics:** page has a single `<h1>` (page title) and `<h2>` per chart card; heatmap cells carry `<title>` for hover/screen-reader context.
