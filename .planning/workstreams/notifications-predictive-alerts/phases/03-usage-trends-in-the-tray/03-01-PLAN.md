<threat_model>
This phase is a LOCAL, READ-ONLY trends feature: it reads a local `~/.claude/usage-history.jsonl`
that the app itself wrote (Phase 2), computes pure aggregations, and renders text rows in a local
GTK menu. No network, no privilege boundary, no untrusted external input, and **no new dependency**
(stdlib `datetime` only — supply-chain surface N/A, no `T-03-SC` install threat). **No high-severity
threats exist.**

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| history file -> in-process reader | `compute_trends` reads a local, app-written JSONL file. The file can be corrupt/partial (killed process, disk error) or unreadable. This is the only boundary; it is not a network or privilege boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | `compute_trends` reading `HISTORY_PATH` | low | mitigate | All reads route through the existing corruption-tolerant `parse_history()` (per-line `json.loads`, numeric-`t` guard), so malformed/partial lines are skipped and trend math never sees garbage (mirrors Phase 2 HIST-03). |
| T-03-02 | Denial of Service | `compute_trends` file read on the poll thread | low | mitigate | Read wrapped in `try/except OSError: return` (degrade to last-known trends, never crash/block); runs in `poll_loop` OFF the Gtk main thread (no UI freeze, D-05); file is bounded by Phase 2's 30-day prune. |
| T-03-03 | Denial of Service | `trend_sparkline` / `trend_burn` auto-scale + averaging | low | mitigate | Explicit `hi == lo` (flat-window) and empty-window/empty-record guards prevent `ZeroDivisionError`; each guard is covered by a `demo()` assert so a regression fails `--selfcheck`. |
</threat_model>
