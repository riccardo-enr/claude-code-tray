<objective>
Build the config data layer -- the actual mechanism CFG-01..05 sit on top of. A tolerant
loader and an atomic writer for `~/.claude/tray-config.json`, a config-driven rewrite of
the `notif_allowed` mute gate Phase 5 left as a seam, and a configurable badge threshold
threaded through the one pure function that reads it.

No menu UI yet -- Plan 2 rides this. That split mirrors Phase 5's shared-path-first
approach: get the mechanism right and provably tolerant before wiring any GTK widget to it.

Purpose: CFG-02 (mute-wins gate semantics, D-04), CFG-03 (persistence), CFG-04
(corruption tolerance, the `260713-fry` bar), and the data-model half of CFG-05
(configurable threshold).
Output: `parse_config` / `load_config` / `save_config`, a config-driven `notif_allowed`,
a threshold-aware `build_label`, and `Monitor.config`.
</objective>

| T-06-01 | Tampering | `parse_config` / `load_config` | **high** | mitigate | A malformed/corrupt/wrong-type `tray-config.json` (hand-edited or half-written by a killed process) must degrade to per-key defaults, never raise -- the same total-tolerance bar `parse_history` already meets (CFG-04, `260713-fry` precedent). | T-06-02 | Denial of Service | `save_config` atomic write | medium | mitigate | An unwritable `~/.claude/` (permissions, disk full, missing dir) must not crash the toggle handler or the Gtk main loop. `save_config` uses the same temp-file + `os.replace` + `except OSError: return` shape as `prune_history`. `self.config` (in-memory) already reflects the toggle regardless of whether the write succeeds, so the running session's behavior stays correct even when persistence silently fails. |
| T-06-03 | Tampering | `usage_threshold` value ingestion | low | mitigate | A `usage_threshold` outside the four presets (hand-edited to `1`, `"90"`, or `999`) must not silently poison the badge comparison. `parse_config` accepts a value only when it is an exact member of `THRESHOLD_CHOICES`; anything else falls back to the default. |
| T-06-SC | Tampering | supply chain | n/a | accept | No package installs in this phase. `json`, `tempfile`, `os` are stdlib, already imported. |
