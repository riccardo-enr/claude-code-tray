## Import-style rule (deliberate, principled split)

- **claude-monitor.py** uses `import core` + `import dashboard` and prefixes EVERY
  reference to a moved symbol as `core.X` / `dashboard.X`. This is the file
  `--selfcheck` cannot exercise (GTK/daemon paths), so a missed rewire must fail as
  a loud, greppable NameError rather than hide.
- **dashboard.py** and **test_claude_monitor.py** use `from core import (...)` /
  `from dashboard import render_dashboard`. Both are fully exercised by `--selfcheck`
  (render_dashboard and demo run inside it), so a wrong import fails the gate
  immediately -- and from-imports keep the moved bodies byte-identical.
- **JS-string caveat:** `_DASH_JS` contains JavaScript `function project(...)`,
  `function hhmm(...)`, and `var WIN5=18000,WIN7=604800` (claude-monitor.py lines
  815, 1004, 1005). These are string content, NOT Python references -- move them
  verbatim, never prefix or import them.
